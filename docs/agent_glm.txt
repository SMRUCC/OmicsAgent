目前我需要你基于我的ollma客户端模块，开发一个能够编写R脚本代码来进行组学数据分析的大语言模型agent。这个agent会需要根据用户的研究主题，设计分析方案，然后编写R脚本代码实现整个组学生物信息学数据分析流程。

这个agent主要是以命令行程序的形式进行运行，需要的输入信息为：

1. research文件：用户以纯文本文件的形式输入其研究的主题以及样本数据的基础信息，例如需要研究什么疾病相关的生物学机制，样本的物种信息，样本的采样组织来源等描述内容，进行差异比对分析的组别比较设计会基于这个文本文件中的描述内容来进行设计
2. expression文件：用户的表达矩阵csv文件的路径，对于单个组学数据分析，直接传递目标矩阵文件的csv文件路径；对于多组学数据分析，则是传递包含有多组学矩阵csv数据文件的文件夹路径。请注意，输入的矩阵文件理论上应该仅支持csv文件。对于用户所输入的表达矩阵csv文件而言，其必须严格的验证和限定表格格式为：行为分子表达数据，列为样本数据。即第一行为样本ID，第一列为分子的ID，其他的单元格都是对应的表达值。
3. annotation文件： 分子注释信息表格，理论上这个表格会提供所有的分子的注释信息，这个表格应该是严格限定为下面的格式的csv文件：id列为分子的ID列信息，与分子表达矩阵中的第一列分子ID一一对应。type列为分子的类别，例如可以为rna（转录组），protein（蛋白质组），metabolite（代谢组），lipid（脂质组）等等。name列则记录了分子的名称，class或者category列记录了分子的分类类别信息。kegg记录了分子的kegg数据库中的id，理论上你可以使用kegg这一列信息用于后续的富集分析
4. sampleinfo文件： 用户的样本元数据信息csv文件的路径。同样的，对于单个组学数据分析，其传递目标矩阵文件的csv文件路径。对于多组学数据分析，理论上同样的则是传递包含有多组学矩阵csv数据文件的文件夹路径，此时理论上应该要求样本元数据信息csv文件的文件名应该是保持和表达矩阵文件的文件名一致。假若多组学的表达矩阵的样本维度一致，可以在多个组学的表达矩阵文件都共用同一套样本元数据信息，则此时也可以仅输入公用的样本元数据信息的csv表格文件。请注意，输入的矩阵文件理论上应该仅支持csv文件。对于用户所输入的表达矩阵csv文件而言，其必须严格的验证和限定表格格式为必须要包含有下面的几列信息的csv表格文件：ID列为表达矩阵文件中的第一行样本ID，sample_name列为样本在作图的时候需要进行显示的标签文本名称，sample_info列为样本的分组标签。除了这三列是必须要出现的，其他的列都是对样本信息做出具体补充的元数据信息，例如可以通过line列记录每一个样本的品种，菌株等信息，time列可以用于记录样本的采集时间点的时间序列信息，等等之类的。这些可选的元数据信息可以用作为WGCNA生物表型关联分析中的生物学表型信息。
5. 参考文献，这个参考文献信息主要以文件夹路径来提供，在这个文件夹中参考文献主要以txt纯文本文件的形式提供内容。这个信息为可选的输入信息，假若存在的话，说明agent需要参考这里的文献中的分析内容来设计差异比较分析流程，以及后续的生物学机制分析流程。
6. 执行分析的工作区文件夹路径。如果这个输入信息缺失，则默认在表达矩阵文件的来源位置中创建analysis文件夹，在analysis文件夹中进行数据分析

这个agent命令行程序主要以ini文件来配置运行环境，运行在windows操作系统上，这个命令行程序在启动的时候读取一个ini文件，检查下面的工具的路径是否存在：

1. Rscript程序路径
2. wkhtmltopdf程序路径

如果ini文件不存在则在特定位置生成一个ini配置文件，或者ini文件中配置的程序路径不存在，则告诉用户如何配置这个ini文件，然后终止程序运行。

同时，这个ini文件还需要记录下面的配置信息：与大语言模型相关的配置，例如服务的url，大语言模型名称，apikey等，然后检查是否有相关的配置信息，如果没有则提示用户填写配置信息然后终止程序。最后按照配置信息进行检查大语言模型服务是否可用，如果不可用，则提示用户后终止程序。

完成了上面的初始化过程后，执行后续的分析流程。

基础的分析模块有：

1. 总体分析： PCA， PLSDA， OPLSDA，表达矩阵总体F检验，表达矩阵总体多因素ANOVA检验
2. 进行差异分析： limma，默认按照pvalue小于0.05来判断是否显著差异，对于代谢组相关的数据，还需要增加vip值是否大于1这个条件。除非用户在research文件中做特别说明，默认不考虑logFC阈值过滤，而是按照pvalue以及VIP筛选后，对剩余的分子按照logFC绝对值做降序排序，取一定数量的top分子做差异分析结果
3. KEGG生物学功能分析：差异富集以及GSVA分析。
4. 生物性状分析：默认按照MAD值做降序排序取top20000各分子做 WGCNA分析
5. 假若数据为时间序列数据，并且样本数量符合要求，则尝试进行bnlearn动态贝叶斯网络分析
6. 假若样本元数据信息清晰，并且样本数量符合要求，则尝试进行PLS-PM因果路径分析

在上面所生成的流程中，需要保存计算结果为csv文件，导出具体的可视化图，生成的图应该是清晰美观的，可以直接用作为论文插图。需要导出png和pdf图像文件，在图上面所有的图注文本内容都应该是英文的。

最终，基于上面的描述内容总结下来，这个agent理论上的组学数据分析的流程如下：

1. 表达矩阵数据预处理
除非用户在样本数据基础信息描述中标注了特殊情况，例如客户明确某一个具体的表达矩阵文件不需要做预处理。一般情况下，表达矩阵需要首先进行过下列的预处理操作才可以进行后续分析流程：按照行做分子表达数据最小阳性值的一半做缺失值填充，然后进行按照列的总和归一化转化为相对表达量，如果有必要的话，需要针对归一化之后的值做log转换。最后针对归一化的表达矩阵按照行做中位数缩放。

2. 总体样本的PCA/PLSDA/OPLSDA分析
基于PCA分析结果的得分，提取样本在主成分（如PC1和PC2）上的得分，计算各样本到其所属组别质心的加权欧氏距离（以各主成分的方差解释率为权重）作为组内离散度。随后，采用置换检验或非参数检验，判断组内距离是否在统计学上显著小于组间距离（或全局平均距离）。若组内距离显著较小，表明同组样本在主要方差特征上高度聚集，可判定该批次/组别的数据重复性质量良好。假若数据质量不好，则给出警告信息。基于PCA的结果，生成阶段性总结性的描述文本用于最后生成研究报告。需要生成具体结果的csv结果文件以及对应的得分散点图。散点图需要使用具体的颜色，散点形状来标记出样本的分组标签，不同的元数据信息等。需要使用置信度椭圆来圈住分组内的样本。

3.设计差异分析的比对组别
接着根据用户的研究主题设计差异分析的比对组别，这些所设计的比对组别应该是深入的契合用户当前的研究主题当前已知相关的生物学机制的。在这一步需要生成一个阶段性的总结文件，向用户阐述差异比对设计的生物学依据，想要达到的分析目的，与用户的研究主题的生物学机制相关性等，在这里所生成的阶段性总结性的描述文本将会用于最后生成研究报告。

4. LIMMA差异比较分析
然后按照所设计的差异比较设计，在这里会需要使用对数据总体做多因素ANOVA检验，使用limma软件进行总体F检验，使用limma做两两比较差异分析。假若是时间序列数据，则还需要增加额外的差异比较分析：将时间因素作为协变量，消除时间因素做差异分析。完成差异分析后，需要绘制火山图，火山图上需要显示top5差异的分子名称。然后统计上下调的差异分子以及总体的差异分子，采用文氏图可视化不同比较间的差异内容。最后使用热图对差异分子做可视化，差异热图的列应该是样本维度，样本维度需要按照样本分组做排序，差异热图的行应该是分子维度，分子维度需要做层次聚类，并且使用颜色块标记出分子的分类信息。分子维度和样本维度都需要展示出具体的对象名字标签文本。同样的，会需要对差异分析结果做阶段性的总结，生成总结性的txt文本文件，描述得到的差异结果是可以如何在生物学机制上对用户的研究主题做出生物学机制上的解释的，不同的比较之间有什么规律性的东西可以让用户进行关注。

5. 生物功能分析
基于差异分析结果，使用keggid进行富集分析，以及做GSVA分析。对于GSVA分析结果，需要按照相同的组别设计进行差异分析。对于富集分析，需要绘制富集结果条形图，在条形图中需要针对通路按照KEGG大分类分组。对于GSVA分析结果，则需要对所有样本的通路得分绘制总体热图，在热图中，列为样本维度，行为kegg通路维度，同样的对于KEGG通路维度，需要按照对应的kegg大分类进行分组。对于样本列维度需要按照样本分组做排序，对于KEGG通路维度，需要针对每一个大类别进行层次聚类，然后针对每一个大类别在热图左侧绘制出对应的层次聚类树。GSVA的差异分析结果，需要绘制火山图，得分图之类的结果图。在这里，会基于生物学通路分析的结果来讲解分析的结果是如何在生物学功能层面上支持用户的研究主题的，将这里阶段性的总结文本同样也生成具体的txt文本文件。

6. 生物学性状关联分析
然后根据用户的研究主题以及结合样本的分组信息，以及样本的元数据信息，构建WGCNA的生物表型关联的性状数据

7. 整理结果文件
最后，生成结果表格。在这一步中，将分析得到的多个中间结果csv表格文件，按照分析的内容主题，写入到对应的xlsx文件之中。所生成的xlsx文件应该是具有如下所描述的样式的：整个表格的字体应该是全局采用Cambria Math 11号字体，表格缩放90%，背景色为默认的白色。第一列理论上应该是id列，采用浅灰色背景色，斜体，黑色字体颜色；第一行为表格的注释说明文本行，需要在第一行中描述清楚这个表格中有哪些内容，每一列的具体含义是什么说清楚，讲明白用户可以从这个表格中获取到哪些生物学知识，这一行的样式为：默认背景色，草绿色字体颜色。第二行则是列标题，样式为：深蓝色背景色，白色字体颜色，加粗字体。需要将第一列+第二行进行freeze panes冻结。所生成的结果xlsx表格文件的文件名，以及表格中的注释文本，表格标题，列标题等所有文本信息应该都是英文的。

8. 撰写论文初稿
针对每一步做出的总结文件和生成的表格以及插图，以论文的形式撰写分析结果报告，以A3大小编写html文件，然后再使用wkhtmltopdf工具转换为PDF文件。
