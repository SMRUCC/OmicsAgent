## 用户需求

将 `Knowledge\KnowledgeBaseBuilder.vb` 中 `GeneratePmcFetchScript` 函数内嵌的 Python 代码提取为独立模板文件，放到 `G:\OmicsWorks\agent\python` 目录中；并将该函数的实现逻辑改为：通过分析上下文环境（`_context.PythonDir`）定位该模板文件，读取文件内容后做字符串替换来生成所需的 Python 脚本。

## 产品概述

对现有知识库构建模块进行内部重构，将硬编码在 VB.NET 函数中的 Python 脚本迁移为外部模板文件，与既有的 NCBI 检索脚本模板（`ncbi_search_template.py`）保持一致的工程模式。

## 核心功能

- 提取 `GeneratePmcFetchScript` 的 Python 代码，生成模板文件 `pmc_fetch_template.py`，保留 `{KNOWLEDGE_DIR}` 占位符。
- 模板文件命名不与 `agent\python` 现有脚本（`ncbi_search_template.py`、`pubmed_search.py`）冲突，不覆盖任何已有文件。
- 改写 `GeneratePmcFetchScript`，改为读取模板文件并通过字符串替换 `{KNOWLEDGE_DIR}` 生成最终脚本，行为与原实现完全一致。
- 调用方 `FetchPmcFullText()` 及脚本执行流程保持不变。

## 技术栈选择

- 语言：VB.NET（.NET，既有项目 OmicsAgent.vbproj）
- 模板目录：复用 `_context.PythonDir`（指向 `G:\OmicsWorks\agent\python`，由 `Config\AgentConfig.vb` 的 `PythonScriptsDir` 与 `Program.vb` 注入）
- 文件读写：复用既有 `PathUtils.ReadAllText` / `File.WriteAllText`（与 `GenerateNcbiSearchScript` 一致）

## 实现方案

**策略**：完全复刻既有 `GenerateNcbiSearchScript` 的模板模式——拼接 `_context.PythonDir` 下的模板路径，检查文件存在性（不存在抛 `FileNotFoundException`，提示风格与现有 NCBI 模板一致），读取模板文本后通过 `.Replace("{KNOWLEDGE_DIR}", outputDir)` 完成占位符替换并返回。

**关键技术决策**：

- 模板文件名采用 `pmc_fetch_template.py`，与现有 `ncbi_search_template.py` 命名约定一致，且不与 `pubmed_search.py`、`ncbi_search_template.py` 重名，满足"不覆盖现有文件"的约束。
- 移除原 `StringBuilder` 逐行拼装逻辑，改为纯文本读取 + 单次字符串替换，降低维护成本并保证生成的 Python 文本与模板文件字节级一致（避免 VB 字符串拼接导致的细微转义差异）。
- 占位符沿用原函数的 `{KNOWLEDGE_DIR}`，无需改动调用方 `FetchPmcFullText()`。

**性能与可靠性**：模板读取为一次性小文件 IO（O(n)），无性能瓶颈；文件缺失时显式抛错，避免静默生成空脚本。错误提示包含完整模板路径，便于排查。

## 实现要点

- 模板文件需保留原始 Python 语义，`KNOWLEDGE_DIR = r'{KNOWLEDGE_DIR}'` 中的原始字符串 `r''` 与 `{KNOWLEDGE_DIR}` 占位符必须原样保留。
- 第 236 行原始 Python 为 `f.write('\n\n=== PMC Full Text (PMCID: %s) ===\n' % pmc_id)`，提取时应保持该字面量（模板中 `\n` 为 Python 字面换行，不做 VB 层转义处理）。
- 改写后的函数去除 `Imports`/StringBuilder 依赖，仅依赖 `System.IO` 与 `PathUtils`（项目已引入）。
- 不改动 `FetchPmcFullText()` 中写入 `fetch_pmc.py` 与 `shell.run_python("scripts/fetch_pmc.py")` 的逻辑，保证向后兼容。

## 架构设计

本次为局部重构，不引入新架构。数据/调用流保持原样：
`FetchPmcFullText()` → `GeneratePmcFetchScript(KnowledgeDir)` 现在改为 `读取 _context.PythonDir/pmc_fetch_template.py` → `.Replace("{KNOWLEDGE_DIR}", KnowledgeDir)` → 返回脚本字符串 → 写入 `ScriptsDir/fetch_pmc.py` → `shell.run_python` 执行。

## 关键代码结构

改写后的 `GeneratePmcFetchScript` 函数签名（接口级定义）：

```
''' <summary>
''' 读取 PMC 全文获取 Python 模板脚本，替换 {KNOWLEDGE_DIR} 占位符后返回最终脚本内容。
''' </summary>
Private Function GeneratePmcFetchScript(outputDir As String) As String
    Dim templatePath = Path.Combine(_context.PythonDir, "pmc_fetch_template.py")
    If Not File.Exists(templatePath) Then
        Throw New FileNotFoundException($"未找到 PMC 全文获取 Python 模板脚本：{templatePath}。请确认 agent/python 目录下存在 pmc_fetch_template.py。")
    End If
    Dim template = PathUtils.ReadAllText(templatePath)
    Return template.Replace("{KNOWLEDGE_DIR}", outputDir)
End Function
```
